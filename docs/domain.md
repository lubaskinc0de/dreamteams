# DreamTeams - Бизнес-требования

## 1. Назначение проекта

**DreamTeams** — веб-сервис для:
- агрегирования хакатонов, олимпиад и других соревновательных мероприятий
- поиска и формирования команд
- взаимодействия организаторов и участников

### 1.1 Целевая аудитория

- **Участники** - студенты, разработчики, дизайнеры, аналитики, желающие участвовать в хакатонах
- **Организаторы** - компании, университеты, IT-сообщества, проводящие хакатоны

## 2. Роли пользователей

### 2.1 Участник

Физическое лицо, участвующее в мероприятиях. Может создавать команды, вступать в существующие команды, регистрироваться на мероприятия.

**Ключевые характеристики:**
- Профессиональные навыки (например: Python, React, UI/UX, DevOps)
- Опыт участия в хакатонах
- Предпочитаемая роль в команде (разработчик, дизайнер, аналитик)

### 2.2 Организатор

Юридическое лицо или официальное сообщество, размещающее мероприятия на платформе.

**Ключевые характеристики:**
- Название организации
- Контактная информация
- История проведенных мероприятий
- Рейтинг (на основе отзывов участников)

### 2.3 Администратор

Управляет платформой, модерирует контент, анализирует метрики.

## 3. Функциональные требования

### 3.1 Регистрация и профили

#### 3.1.1 Участник

**Регистрация:**
- Email + пароль или OAuth (Google, GitHub)
- Подтверждение email

**Профиль:**
- ФИО
- Фотография профиля
- О себе (краткое описание)
- **Навыки** (теги, например: Python, React, Figma)
- Ссылки на портфолио (GitHub, Behance, LinkedIn)
- Предпочитаемая роль
- История участия в мероприятиях
- Рейтинг

**Функции:**
- Просмотр профиля
- Редактирование профиля
- Управление навыками

#### 3.1.2 Организатор

**Регистрация:** ✅ *Реализовано*
- Партнерская модель (доступ выдается администратором или через заявку)
- Поля при регистрации:
  - Название организации
  - Контактный телефон
  - Контактный email
  - Логотип организации (опционально)

**Профиль организации:** ✅ *Частично реализовано*
- Название организации
- Контактная информация (телефон, email)
- Логотип
- Описание организации
- Список созданных мероприятий
- Рейтинг организатора

**Функции:**
- **Просмотр профиля** ✅ *Реализовано*
- Редактирование профиля
- Управление мероприятиями

### 3.2 Агрегатор мероприятий

**Список мероприятий:**
- Отображение всех актуальных мероприятий
- Фильтрация по:
  - дате проведения
  - формату (онлайн/офлайн/гибрид)
  - тематике (AI/ML, веб-разработка, дизайн и т.д.)
  - организатору
  - статусу (регистрация открыта/закрыта)

**Карточка мероприятия:**
- Название
- Описание
- Организатор (ссылка на профиль)
- Даты (начало регистрации, дедлайн регистрации, даты проведения)
- Формат (онлайн/офлайн)
- Локация (для офлайн)
- Призовой фонд
- Требования к участникам
- Статус регистрации
- Кнопка "Зарегистрироваться"

**Регистрация на мероприятие:**
- Индивидуальная регистрация
- Регистрация командой
- Подтверждение регистрации

### 3.3 Система команд

**Создание команды:**
- Создатель = капитан команды
- Название команды
- Описание
- Требуемые навыки (например: "Ищем frontend-разработчика")
- Максимальный размер команды
- Привязка к конкретному мероприятию

**Поиск команды:**
- Просмотр списка команд для конкретного мероприятия
- Фильтрация по требуемым навыкам
- Заявка на вступление в команду

**Поиск участников:**
- Просмотр профилей участников
- Фильтрация по навыкам
- Приглашение в команду

**Управление командой:**
- Просмотр состава
- Принятие/отклонение заявок
- Исключение участников (только капитан)
- Передача капитанства

### 3.4 Размещение мероприятий (для организаторов)

**Создание мероприятия:**
- Название
- Описание
- Тематика
- Формат (онлайн/офлайн/гибрид)
- Локация (для офлайн)
- Даты:
  - начало регистрации
  - окончание регистрации
  - дата начала мероприятия
  - дата окончания мероприятия
- Требования к участникам
- Призовой фонд
- Партнеры
- Правила участия
- FAQ

**Управление мероприятием:**
- Редактирование информации
- Просмотр списка зарегистрированных участников
- Просмотр списка команд
- Экспорт данных (CSV, JSON)
- Публикация результатов
- Архивация мероприятия

### 3.5 Система рейтинга

**Рейтинг участников:**
- Рассчитывается на основе:
  - количества участий в мероприятиях
  - занятых мест
  - отзывов от сокомандников
- Отображается в профиле

**Рейтинг организаторов:**
- Рассчитывается на основе:
  - отзывов участников
  - количества проведенных мероприятий
  - средней оценки мероприятий
- Отображается в профиле организации

**Отзывы:**
- Участник может оставить отзыв о мероприятии
- Участники могут оставлять отзывы о сокомандниках
- Модерация отзывов

### 3.6 Интеграция и API

**API для организаторов:**
- Получение списка зарегистрированных участников
- Получение списка команд
- Создание мероприятия через API
- Обновление информации о мероприятии
- Webhook для уведомлений о новых регистрациях

**Интеграция с внешними системами:**
- OAuth для авторизации (Google, GitHub)
- Интеграция с календарем (Google Calendar, Outlook)
- Экспорт данных

### 3.7 Уведомления

**Типы уведомлений:**
- Новое мероприятие по интересам
- Приглашение в команду
- Заявка на вступление в команду
- Приближение дедлайна регистрации
- Начало мероприятия
- Публикация результатов

**Каналы:**
- Email
- Push-уведомления (для веб-версии)
- In-app уведомления

## 9. Текущий статус реализации

### 9.1 Реализовано ✅

#### Регистрация организатора
- **Endpoint:** `POST /landlords/`
- **Описание:** Создание аккаунта организатора и профиля организации
- **Поля:**
  - `landlord_name` - название организации (строка, максимум 70 символов)
  - `phone_number` - контактный телефон (российский формат)
  - `contact_email` - контактный email
  - `avatar` - логотип организации (опционально)
- **Процесс:**
  1. Создается новый пользователь в системе
  2. Создается профиль организатора
  3. Профиль привязывается к пользователю
  4. Возвращается `landlord_id` и `user_id`

#### Просмотр профиля организатора
- **Endpoint:** `GET /profile/`
- **Описание:** Получение информации о профиле текущего пользователя
- **Возвращаемые данные:**
  - `user_id` - идентификатор пользователя
  - Информация об организаторе (если роль назначена):
    - `id` - идентификатор организатора
    - `landlord_name` - название организации
    - `phone_number` - контактный телефон
    - `contact_email` - контактный email
    - `avatar` - логотип организации

## 10. Технологический стек

### 10.1 Backend
- **Framework:** FastAPI
- **Language:** Python 3.13+
- **Architecture:** Clean Architecture
- **ORM:** SQLAlchemy (imperative style)
- **Database:** PostgreSQL
- **Cache:** Redis
- **DI:** Dishka
- **Migrations:** Alembic

### 10.2 Аутентификация
- **Identity Provider:** Keycloak
- **Auth Proxy:** OAuth2-Proxy
- **Reverse Proxy:** Nginx
- **Protocol:** OAuth 2.0 / OIDC

### 10.3 Infrastructure
- **Containerization:** Docker, Docker Compose
- **Logging:** Structlog (JSON format)
- **Log aggregation:** Vector → Loki → Grafana
- **CI/CD:** GitHub Actions
- **Container Registry:** GitHub Container Registry

### 10.4 Code Quality
- **Type Checking:** mypy (strict mode)
- **Linting:** ruff
- **Dependency Rules:** import-linter
- **Testing:** pytest (integration tests)
