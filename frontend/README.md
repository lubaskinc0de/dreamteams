# Посуточник - Frontend

Фронтенд-приложение для платформы управления краткосрочной арендой "Посуточник".

## Технологический стек

- **Nuxt 3** - фреймворк для Vue.js
- **TypeScript** - типизация
- **NuxtUI** - UI компоненты
- **Pinia** - управление состоянием
- **Tailwind CSS** - стилизация (встроен в NuxtUI)

## Функциональность

### Основные возможности

1. **Регистрация арендодателя** (`/register`)
   - Форма с валидацией имени (макс. 70 символов)
   - Валидация российского номера телефона
   - Обработка ошибок (уже зарегистрирован, ошибки валидации и т.д.)

2. **Профиль пользователя** (`/profile`)
   - Просмотр данных арендодателя
   - Информация о статусе регистрации
   - Предложение регистрации, если пользователь не является арендодателем

3. **Главная страница** (`/`)
   - Описание платформы
   - Навигация по основным разделам

### API эндпоинты

- `POST /landlords/` - регистрация арендодателя
- `GET /users/me` - получение профиля текущего пользователя

## Установка и запуск

### Установка зависимостей

```bash
npm install
```

### Настройка окружения

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Настройте переменную окружения:

```env
NUXT_PUBLIC_API_BASE=http://localhost:8000
```

### Режим разработки

Запуск dev-сервера на `http://localhost:3000`:

```bash
npm run dev
```

### Production build

```bash
# Сборка приложения
npm run build

# Предварительный просмотр production build
npm run preview
```

## Обработка ошибок

Приложение обрабатывает следующие коды ошибок от API:

- `VALIDATION_ERROR` (422) - ошибки валидации формы
- `UNAUTHORIZED` (401) - пользователь не авторизован
- `AUTH_USER_ALREADY_EXISTS` (409) - пользователь уже зарегистрирован как арендодатель
- `USER_NOT_FOUND` (404) - пользователь не найден
- `ACCESS_DENIED` (403) - доступ запрещен
- `INTERNAL_SERVER_ERROR` (500) - внутренняя ошибка сервера

## Валидация форм

### Регистрация арендодателя

- **Имя арендодателя**:
  - Обязательное поле
  - Максимум 70 символов
  - Валидация при потере фокуса и перед отправкой

- **Номер телефона**:
  - Обязательное поле
  - Российский формат: +7XXXXXXXXXX (11 цифр)
  - Автоматическое форматирование при вводе
  - Валидация при потере фокуса и перед отправкой

## Особенности реализации

1. **Type-safe API клиент** - все запросы типизированы
2. **Централизованная обработка ошибок** - единый формат ошибок
3. **Reactive состояние** - Pinia stores для управления данными
4. **Responsive дизайн** - адаптивная верстка с Tailwind CSS
5. **Валидация на клиенте** - проверка данных до отправки на сервер
